Stage 3: Архитектура обработчиков сообщений и кэширования

Состояния пользователя (FSM)

IDLE (неактивный)
Начальное состояние при старте бота. Пользователь зарегистрирован в системе и может начать взаимодействие.
BROWSING (просмотр)
Пользователь просматривает каталог категорий и товаров.
VIEWING_PRODUCT (просмотр товара)
Пользователь открыл детальное описание товара и может добавить его в корзину.
CART (корзина)
Пользователь просматривает содержимое корзины и может изменить состав или перейти к оформлению.
CHECKOUT (оформление)
Пользователь подтверждает заказ и переходит к оплате.
PAYMENT (оплата)
Заказ ожидает подтверждения платежа в системе.
COMPLETED (завершено)
Заказ успешно оплачен и отправлен на обработку.

Маршрутизация обработчиков

handle_start()
Регистрация пользователя и показ главного меню
/start register_or_get_user
handle_help()
Справка по использованию бота
/help help
handle_catalog()
Получение списка категорий из кэша
cat:all Redis
handle_category()
Получение товаров в категории
cat:{category_id} products list
handle_product()
Показ детальной информации о товаре
prod:{product_id} HTML parse
handle_cart_add()
Добавление товара в корзину
cart:add:{product_id} quantity++
handle_cart_view()
Просмотр содержимого корзины
cart:view total calc
handle_checkout()
Инициирование процесса оформления заказа
checkout:start order create

Форматы Callback Data

📌 Каталог и товары

cat:all Показать все категории
cat:{category_id} Показать товары категории
prod:{product_id} Показать детали товара

🛒 Корзина

cart:add:{product_id} Добавить товар в корзину
cart:view Показать содержимое корзины
cart:clear Очистить корзину полностью

✅ Оформление заказа

checkout:start Начать оформление заказа
checkout:confirm Подтвердить заказ
checkout:cancel Отменить оформление

Поток обработки данных

Пользователь нажимает кнопку в чате Telegram
Hydrogram получает CallbackQuery с callback_data
Обработчик парсит callback_data по формату "prefix:id"
Получение данных из Redis кэша (категории, товары)
Формирование ответа с новой клавиатурой (InlineKeyboardMarkup)

💾 Кэширование в Redis

🔑 Ключ кэша: Все ключи начинаются с префикса "bot:" для изоляции от других сервисов

🗂️ Структура хранилища

bot:categories
Список всех категорий товаров (JSON)
TTL: 3600 секунд (1 час)
bot:products:cat:{category_id}
Товары в конкретной категории (JSON)
TTL: 3600 секунд (1 час)
bot:products:all
Все товары (для быстрого поиска)
TTL: 3600 секунд (1 час)
bot:cart:{user_id}
Корзина пользователя с товарами и суммой
TTL: 86400 секунд (24 часа)

🛤️ Путь пользователя

Сценарий 1: Первое посещение

1. Отправка /start
Пользователь нажимает "Start" в Telegram
2. Регистрация
Функция register_or_get_user() создает новую запись User в БД или находит существующую
3. Главное меню
Показывается InlineKeyboardMarkup с кнопками "Каталог" и "Корзина"

Сценарий 2: Покупка товара

1. Каталог
Пользователь нажимает "Каталог" → handle_catalog() загружает категории из Redis
2. Выбор категории
Нажимает на категорию → handle_category() получает товары этой категории
3. Выбор товара
Нажимает на товар → handle_product() показывает описание и кнопку "Добавить в корзину"
4. В корзину
Нажимает "Добавить в корзину" → handle_cart_add() сохраняет товар в Redis на 24 часа
5. Просмотр корзины
Нажимает "Корзина" → handle_cart_view() показывает сумму и кнопку оформления
6. Оформление
Нажимает "Оформить заказ" → handle_checkout() создает Order и переводит в PAYMENT

🏗️ Архитектура компонентов

BotConfig
Синглтон для получения конфигурации из AppConfiguration (токен, приветствие, справка)
BotCache
Обертка над Redis для кэширования товаров (3600s) и корзин (86400s)
Keyboards
7 функций для построения InlineKeyboardMarkup с правильной маршрутизацией callback_data
Handlers
8 async функций для обработки команд /start, /help и callback-запросов от кнопок
Enums
UserState (FSM), CartAction, ButtonCallback для типобезопасности
